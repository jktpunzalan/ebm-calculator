saliksic.com + EBM Appraisal — Project Instructions

Complete specification for building, deploying, and maintaining the saliksic.com website and EBM Appraisal mobile app using Windsurf, Claude, and Hostinger.

Frontend: React Native (Expo) + HTML/CSS/JS Backend: Laravel 11 Database: MySQL 8.0 + Eloquent ORM Hosting: Hostinger IDE: Windsurf + Claude
Section 1

Project Overview

This project consists of two interconnected products deployed under a single Hostinger account at saliksic.com:

Product 1 — saliksic.com (web)

A public-facing website for a clinical epidemiology research and knowledge translation group. It serves four content areas:

  • Landing page — homepage with site statistics, navigation to all sections, and a featured publications feed.
  • Publications — interactive HTML knowledge translation (KT) pages, each presenting a research article in an audience-adaptive, action-oriented format with view/share tracking, PDF export, and infographic sharing.
  • Journal — a browsable archive of peer-reviewed journal volumes and articles with DOI linking.
  • Professional page — team profiles, credentials, affiliations, and contact.

Product 2 — EBM Appraisal App (React Native)

A mobile-first clinical tool for evidence-based medicine with three modules:

  • Critical appraisal — 8 study type modules (Therapy, Diagnosis, Harm, Prognosis, Systematic Review, Economic Evaluation, Clinical Practice Guidelines, Screening), each with a structured checklist and live calculators.
  • Study library — persistent storage of all completed appraisal results, filterable and exportable.
  • Individualization — patient-specific calculation of NNT, ARR, post-test probability, etc., using saved study results and user-entered baseline risk.
Shared Laravel backend

Both products share a single Laravel 11 API and MySQL database on Hostinger. The web frontend and React Native app both authenticate via Laravel Sanctum tokens and call the same /api/ routes defined in routes/api.php.

Database — Laravel Migrations

All database changes are managed through Laravel migrations, not raw SQL files. Each table has its own timestamped migration file in database/migrations/. Run php artisan migrate on Hostinger via SSH to apply them.

Windsurf prompt to generate all migrations

"Generate Laravel 11 migration files for the saliksic.com project. Create one migration per table: users (with role ENUM: admin, editor, clinician), appraisals, appraisal_results, study_library, individualizations, publications, pub_statistics, pub_events, journal_articles, professional_items, content_pages. Use foreignId()->constrained() for foreign keys. Add indexes on slug (publications), user_id (appraisals, library, individualizations), and publication_id (pub_statistics, pub_events). Include timestamps on all tables."

Key Eloquent patterns to use

// Appraisal model — JSON cast, relationships, scope
class Appraisal extends Model {
    protected $fillable = ['user_id','study_type','title','authors','year','checklist','notes','validity_score'];
    protected $casts = ['checklist' => 'array'];  // auto JSON encode/decode

    public function user(): BelongsTo { return $this->belongsTo(User::class); }
    public function results(): HasMany { return $this->hasMany(AppraisalResult::class); }
    public function scopeForUser($q, $userId) { return $q->where('user_id', $userId); }
}

// Publication model — record events, update stats atomically
class Publication extends Model {
    protected $casts = ['audience_tags' => 'array', 'published_at' => 'datetime'];

    public function stats(): HasOne { return $this->hasOne(PubStatistic::class); }
    public function recordView(): void {
        $this->events()->create(['event_type' => 'view', 'ip_hash' => hash('sha256', request()->ip())]);
        $this->stats()->increment('views');  // atomic DB increment — no race condition
    }
}

API Endpoints — Laravel routes/api.php

All routes are defined in routes/api.php. Protected routes use Laravel Sanctum's auth:sanctum middleware. Admin routes additionally use a custom admin middleware that checks the user's Spatie role.

// routes/api.php — structure overview
Route::post('/auth/register', [AuthController::class, 'register']);
Route::post('/auth/login',    [AuthController::class, 'login']);

Route::middleware('auth:sanctum')->group(function () {
    Route::post('/auth/logout', [AuthController::class, 'logout']);

    Route::apiResource('appraisals', AppraisalController::class);
    Route::apiResource('library', LibraryController::class)->only(['index','store','destroy']);
    Route::apiResource('individualization', IndividualizationController::class)->only(['index','store']);

    Route::middleware('role:admin')->prefix('admin')->group(function () {
        Route::get('/stats', [AdminStatsController::class, 'index']);
        Route::apiResource('publications', AdminPublicationController::class);
    });
});

// Public routes (no auth)
Route::get('/publications',               [PublicationController::class, 'index']);

Route::post('/publications/{slug}/view',  [PublicationController::class, 'recordView']);

Publications — KT HTML Pages

Each publication is a self-contained HTML file stored via Laravel Storage at storage/app/public/publications/[slug].html, symlinked to public/storage/. Publications are authored using Claude and uploaded via the admin panel, which calls POST /api/admin/publications to store the file and register metadata in MySQL.

Required page structure (every publication)

  1. Sticky top nav with progress bar, saliksic.com brand, section links, Share button.

  2. Hero block — type chips, H1 title, authors, DOI.

  3. Audience toggle — Educator / Student / Policymaker / Community leader. Rewrites bottom line and action section.

  4. Bottom line card — dark background, audience-specific sentence, keyword pills.

  5. Implementation section FIRST — accordion action list per audience with expandable detail.

  6. Evidence section below — stat grid, bar charts, framed as "Evidence behind the recommendations."

  7. Framework explorer, Barriers, Evidence quality, Go deeper — in that order.

  8. Sticky share bar — view/share/save stats, Copy link, PDF, Share KT modal with infographic and channel selection.

Statistics integration script (included in every publication HTML)

const SLUG = 'punzalan-2025-ceme-kt';  // set per file
const API  = 'https://api.saliksic.com';

(async () => {
  await fetch(`${API}/api/publications/${SLUG}/view`, { method: 'POST' });
  const res  = await fetch(`${API}/api/publications/${SLUG}/stats`);
  const { data } = await res.json();
  document.getElementById('stat-views').textContent  = data.views.toLocaleString();
  document.getElementById('stat-shares').textContent = data.shares.toLocaleString();
  document.getElementById('stat-saves').textContent  = data.saves.toLocaleString();

Sanctum token authentication flow

// services/api.ts — axios instance with Sanctum token
import axios from 'axios';
import AsyncStorage from '@react-native-async-storage/async-storage';

const api = axios.create({ baseURL: 'https://api.saliksic.com/api' });

api.interceptors.request.use(async (config) => {
  const token = await AsyncStorage.getItem('sanctum_token');
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// On login — store the token Laravel Sanctum returns
const { data } = await api.post('/auth/login', { email, password });
await AsyncStorage.setItem('sanctum_token', data.token);

Hostinger Deployment — Laravel

Critical: Laravel document root on Hostinger

The Laravel project root must live outside public_html/. Only the Laravel public/ folder is exposed to the web. Point api.saliksic.com to /home/[user]/saliksic-api/public/, not to the Laravel root. This prevents exposing .env, vendor/, and other sensitive files.

Hostinger plan

Use Business Web Hosting or Cloud Startup. Both support PHP 8.2, MySQL 8.0, SSH access, Composer, and custom document roots per subdomain.

Initial deployment checklist

  1. Domain and subdomains — in Hostinger panel: saliksic.com → public_html/. api.saliksic.com → /home/[user]/saliksic-api/public/. app.saliksic.com → public_html/app/.

  2. SSL — enable Let's Encrypt for all three subdomains in Hostinger SSL panel.

  3. PHP version — set to PHP 8.2 in Hostinger PHP Configuration panel.

  4. Upload Laravel project — upload the Laravel project folder to /home/[user]/saliksic-api/ via FTP or SSH git clone. Do NOT put it inside public_html/.

  5. Create MySQL database — in Hostinger Databases panel, create database and user. Note host, database name, username, password.

  6. Configure .env — upload .env directly to /home/[user]/saliksic-api/ via File Manager. Set all values:

APP_NAME=saliksic-api
APP_ENV=production
APP_KEY=                        # generated by artisan key:generate
APP_DEBUG=false
APP_URL=https://api.saliksic.com

DB_CONNECTION=mysql
DB_HOST=localhost
DB_PORT=3306
DB_DATABASE=your_db_name
DB_USERNAME=your_db_user
DB_PASSWORD=your_db_password

SANCTUM_STATEFUL_DOMAINS=saliksic.com,app.saliksic.com

FILESYSTEM_DISK=public         # for Storage::disk('public')

ADMIN_PASSWORD=                # used by AdminUserSeeder

  1. SSH into Hostinger and run the Laravel setup commands:

cd ~/saliksic-api

composer install --no-dev --optimize-autoloader
php artisan key:generate
php artisan migrate --force
php artisan db:seed --class=AdminUserSeeder
php artisan storage:link          # creates public/storage → storage/app/public symlink
php artisan config:cache
php artisan route:cache
php artisan view:cache

  1. Test the API — call https://api.saliksic.com/api/publications in a browser. It should return a JSON response.

  2. Deploy web frontend — upload index.html, assets/, and initial publication HTML files to public_html/.

  3. React Native build — run npx expo export --platform web locally. Upload the dist/ output to public_html/app/. For native app stores, use Expo EAS Build.

Ongoing deployment

# After pushing code changes via git
cd ~/saliksic-api
git pull origin main
composer install --no-dev --optimize-autoloader
php artisan migrate --force     # only if new migrations exist
php artisan config:cache
php artisan route:cache
php artisan view:cache
Never commit .env to Git

Add .env to .gitignore on day one. The .env.example file is committed with placeholder values. The real .env is uploaded directly to Hostinger via File Manager and stored in a password manager.

CORS configuration for cross-origin requests

// config/cors.php
'paths' => ['api/*'],
'allowed_origins' => [
    'https://saliksic.com',
    'https://app.saliksic.com',
    'http://localhost:8081',    // Expo dev server
],
'allowed_methods' => ['*'],
'allowed_headers' => ['*'],
'supports_credentials' => false,
Section 15

Development Phases

Build in this order — each phase is deployable

Never move to the next phase without deploying the previous one to Hostinger and testing it on a real device. Laravel's Artisan CLI makes this fast: scaffold → migrate → test → deploy.

Phase 1
1–2 weeks
Laravel foundation
  • Laravel 11 installed, Sanctum + Spatie Permissions configured
  • All migrations created and run on Hostinger
  • AuthController: register, login (returns Sanctum token), logout
  • CORS configured for saliksic.com + Expo dev
  • Admin user seeded — confirm login works via curl
  • Git repository initialised, .env secured and gitignored
Phase 2
2–3 weeks
EBM Appraisal app core
  • Appraisal, Library, Individualization Laravel controllers + Form Requests + API Resources
  • React Native Expo app with bottom tab navigation
  • All 8 appraisal screens with checklists and calculators (offline JS logic)
  • Sanctum token stored in AsyncStorage, sent with all API calls
  • Save/load appraisals, library, and individualization log — tested on iOS + Android via Expo Go
Phase 3
2 weeks
saliksic.com web — landing + publications
  • Landing page live at saliksic.com, publications feed fetches from Laravel API
  • First 3 publications authored in Claude and uploaded via admin panel
  • PublicationController: view tracking, stats, DOMPDF PDF generation
  • Share bar showing live counts, PDF export, share modal functional
  • Responsive design tested on mobile — Laravel Storage symlink verified
Phase 4
1–2 weeks
Journal, Professional, Admin panel
  • Journal archive page live with initial articles
  • Professional page live with team profiles
  • Alpine.js admin panel: publication upload, stats dashboard, user role management
  • Admin routes protected by Sanctum + Spatie admin role check
Phase 5
Ongoing
Content, polish, app store
  • Expo EAS Build for iOS App Store and Google Play submission
  • Add publications regularly using the Claude KT template
  • SEO: meta tags, Open Graph, sitemap.xml, robots.txt
  • Laravel config/route/view caching on every deploy for performance
  • Monitor pub_statistics for most-shared content to guide future publications
